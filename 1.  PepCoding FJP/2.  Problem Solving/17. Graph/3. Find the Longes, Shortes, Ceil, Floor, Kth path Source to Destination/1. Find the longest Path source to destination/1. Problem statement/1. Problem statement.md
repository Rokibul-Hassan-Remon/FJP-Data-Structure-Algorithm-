# ✅ **Longest Path in a Graph — Problem Statement (Clean Version)**

**Problem:**  
You are given an **undirected weighted graph** represented by `V` vertices and `E` edges.  
Each edge has a weight.

You are also given two vertices:

- **source vertex (src)**
    
- **destination vertex (dest)**
    

You must find:

### **➡️ The longest simple path (no node repeated) from `src` to `dest`.**

and also return the **total weight** of that longest path.

A simple path means:

- কোনও vertex একবারের বেশি visit করা যাবে না
    
- cycle allow করা যাবে না
    
- সব path explore করতে হবে (backtracking)
    

---

## **Input Format**
```css
V                       // number of vertices
E                       // number of edges
v1 v2 weight            // E lines: each edge
...
src                     // starting vertex
dest                    // ending vertex
```

---

## **Output Format**
```css
LongestPath: <list-of-vertices>
TotalWeight: <sum-of-weights>
```

---

## **Constraints**

- 1 ≤ V ≤ small/medium (DFS/backtracking possible)
    
- Graph may contain cycles
    
- You must consider only **simple paths**
    

---

## **Goal Explanation (Bangla)**

তুমি মূলত খুঁজবে:

### **src → dest** যাবার সব possible simple path

এবং

### যেটার total weight সবচেয়ে বেশি, সেটাই longest path.

---

## **Example**

**Input:**
```css
5
6
0 1 5
0 2 3
1 3 8
2 3 2
1 4 4
3 4 6
0
4

```

**Possible Paths:**

- 0 → 1 → 4 (weight = 9)
    
- 0 → 2 → 3 → 4 (weight = 11)
    
- 0 → 1 → 3 → 4 (weight = 19) ✔️ (Longest)
    

**Output:**
```css
LongestPath: 0 1 3 4
TotalWeight: 19
```
